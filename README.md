# bolt-rest-assured

REST Assured API test suite for Real Brokerage bolt microservices.

## Stack

- Java 17
- Maven 3.9+
- JUnit 5
- REST Assured 5.5.1

## Setup

1. Clone the repo:
   ```bash
   git clone https://github.com/manepranal/bolt-rest-assured.git
   cd bolt-rest-assured
   ```

2. Set your auth token in the relevant config file:
   ```
   src/test/resources/config-team1.properties
   src/test/resources/config-team2.properties
   src/test/resources/config-staging.properties
   src/test/resources/config-play.properties
   ```
   Update `auth.token=YOUR_AUTH_TOKEN_HERE` with a valid Bearer token.

3. Install dependencies:
   ```bash
   mvn install -DskipTests
   ```

## Running Tests

Run all tests against an environment:
```bash
mvn test -Pteam1
mvn test -Pteam2
mvn test -Pstaging
mvn test -Pplay
```

Run a specific service tag:
```bash
mvn test -Pteam1 -Dgroups=arrakis
mvn test -Pteam1 -Dgroups=yenta
mvn test -Pteam1 -Dgroups=sherlock
```

Run a specific test class:
```bash
mvn test -Pteam1 -Dtest=YentaSherlockAtomicityTest
```

## Adding New Tests

New test classes are auto-generated by the [YouTrack QA Agent](https://github.com/manepranal/youtrack-qa-agent):

```bash
qa <TICKET-ID> --env team1 --pr
```

This fetches the ticket, generates the test class, and raises a PR automatically.

## Environments

All environments follow the same URL pattern — just swap in the env name:

| Env | Base URL pattern |
|-----|-----------------|
| team1 | `https://{service}.team1realbrokerage.com` |
| team2 | `https://{service}.team2realbrokerage.com` |
| team3 | `https://{service}.team3realbrokerage.com` |
| team4 | `https://{service}.team4realbrokerage.com` |
| staging | `https://{service}.stagerealbrokerage.com` |
| play | `https://{service}.playrealbrokerage.com` |

Each env has its own config file at `src/test/resources/config-{env}.properties` with all service base URLs and an `auth.token` property.

> Example for `yenta` on `team3`:
> `https://yenta.team3realbrokerage.com`

To add a new environment, duplicate an existing config file and update the token:
```bash
cp src/test/resources/config-team1.properties src/test/resources/config-team3.properties
# then update auth.token inside config-team3.properties
```
